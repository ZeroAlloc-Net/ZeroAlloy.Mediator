# ZMediator Design Document

## Overview

ZMediator is a zero-allocation mediator library for .NET 10. It uses a Roslyn incremental source generator to wire all dispatch at compile time — no reflection, no dictionaries, no virtual dispatch, no delegate allocation per request.

## Goals

- True zero-allocation per request dispatch
- Full mediator pattern: request/response, notifications, streaming, pipeline behaviors
- Compile-time safety: missing or duplicate handlers are build errors
- DI-agnostic: factory delegates for handler dependencies, no container required
- Source generator only: no runtime assembly scanning

## Non-Goals

- Exception handler pipeline (use pipeline behaviors instead)
- Polymorphic dispatch
- Pre/post processors as separate concepts
- MS DI integration (can be added later as separate package)

## Core Abstractions

All in the `ZMediator` package. Interfaces and attributes only — no logic, no allocations.

## Source Generator Output

The generator emits a static `Mediator` class with strongly-typed overloads per request/notification/stream type.

### Request Dispatch

Each request type gets its own `Send` overload. Pipeline behaviors are inlined as nested static lambda calls:

```csharp
// <auto-generated/>
public static partial class Mediator
{
    private static Func<PingHandler>? _pingHandlerFactory;

    public static ValueTask<string> Send(Ping request, CancellationToken ct = default)
    {
        return LoggingBehavior.Handle<Ping, string>(
            request, ct, static (r, c) =>
                ValidationBehavior.Handle<Ping, string>(
                    r, c, static (r2, c2) =>
                    {
                        var handler = _pingHandlerFactory?.Invoke() ?? new PingHandler();
                        return handler.Handle(r2, c2);
                    }));
    }
}
```

### Notification Dispatch

Sequential by default:

```csharp
public static async ValueTask Publish(UserCreated notification, CancellationToken ct = default)
{
    var h1 = _sendWelcomeEmailFactory?.Invoke() ?? new SendWelcomeEmailHandler();
    await h1.Handle(notification, ct);
}
```

Parallel when `[ParallelNotification]` is applied to the notification type:

```csharp
public static ValueTask Publish(OrderPlaced notification, CancellationToken ct = default)
{
    var h1 = _orderAnalyticsFactory?.Invoke() ?? new OrderAnalyticsHandler();
    var h2 = _orderEmailFactory?.Invoke() ?? new OrderEmailHandler();
    return new ValueTask(Task.WhenAll(
        h1.Handle(notification, ct).AsTask(),
        h2.Handle(notification, ct).AsTask()));
}
```

### Stream Dispatch

```csharp
public static IAsyncEnumerable<int> CreateStream(
    CounterStreamRequest request, CancellationToken ct = default)
{
    var handler = _counterStreamFactory?.Invoke() ?? new CounterStreamHandler();
    return handler.Handle(request, ct);
}
```

## Pipeline Behavior Composition

Behaviors are sorted by `Order` attribute. Scoped behaviors (with `AppliesTo`) interleave with global behaviors. The generator inlines the full chain as nested calls — no list, no loop, no allocation.

Example ordering for `CreateUser` with:
- LoggingBehavior (global, order=0)
- CreateUserAuditBehavior (scoped to CreateUser, order=0)
- ValidationBehavior (global, order=1)

Result: `Logging → Audit → Validation → CreateUserHandler.Handle()`

Same-order behaviors: global before scoped.

## Zero-Allocation Techniques

| Technique | Avoids |
|---|---|
| `ValueTask<T>` return types | `Task<T>` heap allocation for sync completions |
| `readonly record struct` for requests/Unit | Class heap allocation |
| Static method dispatch | Virtual dispatch overhead |
| Compile-time method overloads | `Dictionary<Type, Handler>` lookups |
| Inlined pipeline chain | Delegate chain allocation |
| `static` lambdas in pipeline | Closure captures |
| Direct handler type in generated code | `Activator.CreateInstance` / reflection |

## Analyzer Diagnostics

| ID | Severity | Description |
|---|---|---|
| ZM001 | Error | Request type has no registered handler |
| ZM002 | Error | Request type has multiple handlers |
| ZM003 | Warning | Request type is a class — recommend `readonly record struct` |
| ZM004 | Error | Handler method signature doesn't match expected pattern |
| ZM005 | Error | Pipeline behavior missing static `Handle<TRequest, TResponse>` method |
| ZM006 | Warning | Duplicate `[PipelineBehavior(Order)]` values — ambiguous ordering |
| ZM007 | Error | Stream handler returns `Task`/`ValueTask` instead of `IAsyncEnumerable` |

## User-Facing API Example

```csharp
// Define request
public readonly record struct Ping(string Message) : IRequest<string>;

// Define handler
public class PingHandler : IRequestHandler<Ping, string>
{
    public ValueTask<string> Handle(Ping request, CancellationToken ct)
        => ValueTask.FromResult($"Pong: {request.Message}");
}

// Use
var result = await Mediator.Send(new Ping("Hello"), ct);

// With factory for dependencies
Mediator.Configure(cfg =>
{
    cfg.SetFactory<CreateUserHandler>(() => new CreateUserHandler(myDbContext));
});
```
